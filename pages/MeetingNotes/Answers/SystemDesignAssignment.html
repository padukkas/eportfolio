System Design Assignment

Question:

You have been asked to design a new software system to manage self- service checkouts at a well-known high street supermarket chain. When an item is scanned, it is added to the customers purchases in a virtual shopping basket. Items can either be scanned with the barcode reader or they can be weighed using the integrated scales. When the customer has finished scanning their items, they will be given the option to scan their loyalty card, if they have one. The customer then purchases their items using several different payment methods (e.g., cash, card, ApplePay, AndroidPay, vouchers, loyalty points). Supermarket staff will have the ability to override transactions where required, for example if a price has been scanned incorrectly, they will also be responsible for checking the age of a customer for restricted items. A completed transaction will update the supermarkets stock control systems and generate an alert for warehouse staff if either the shelves should be replenished or if stock needs to be reordered. Using the scenario provided you are required to generate an object model for the proposed system.

Class Diagram:

To initiate any transaction there is a “Customer” class, which contains multiple attributes and an operation. Customer has a direct association to the “Checkout Scanner” class. The “Customer” class also has a composite relationship with the “Loyalty Card” class which allows customers to collect points from instore purchases. The relationship between these two classes is composition because “Loyalty Card” would not exist without a customer.

“Checkout Scanner” class which allows the customer to scan/weigh item or scan loyalty card also has an aggregate relationship with “Virtual Shopping Cart” class because without the shopping cart the “Checkout Scanner” can exist just not used. The Scanner class has an association with the “Loyalty Card” class since the same scanner is used to scan the loyalty card.

“Virtual Shopping Cart” class which holds the items until the customer makes the payment has a composite relationship with the “Item” class since the shopping cart would not exist without items. When items are added to the shopping cart there is the option to request assistance to override items or for the restricted items, these are stored as attributes in the “Item” class. “Virtual Shopping Cart” class has a direct associate with the “Payment processor” class, since after all the items are added to cart the next step would be to process the payment.

While “Item” class which contains the item information requires a composite relationship with the “Virtual Shopping Cart” class, it has an associate relationship with the “Inventory System” class, as an item is added to a shopping cart it will keep count so the system can know when that item runs out of stock in the shelves.

The “Inventory System” class has a direct associate relationship with the “Payment Processor” class since when the payment goes through that item has to be deducted from the inventory. The “Inventory system” class also has an aggregate relationship with the “Warehouse” to “notify” the warehouse once the items are all out of stock in the entire store which would trigger the “Warehouse Staff” class to “reorder” the item to the warehouse.

“Payment Processor” class has a composite relationship with the “Payment Method” class since the payment method would not exists without the payment processor. The “Payment Method” class has direct association to all the methods of payment types such as Cash, Credit Card, Apple pay and etc. “Apple pay” and “Android pay” uses inheritance from credit card as credit card information that is being added to those devices. Customer can pay with their Loyalty points therefore “Loyalty Points” class uses inheritance from “Loyalty Card” class.

There are two Staff classes, “Warehouse Staff” and “Floor Staff”. “Warehouse Staff” has an aggregate relationship with the “Warehouse” class, since the warehouse can exist without the warehouse staff. “Warehouse staff” class also has a direct association to the “Item” class as the Warehouse Staff “restocks” items once they run out said item on the shelves. The “Floor Staff” class has a direct association with the “Virtual Shopping Cart” class, as the floor staff has to override items in the shopping cart and to handle customers when they add restricted items to the cart.
