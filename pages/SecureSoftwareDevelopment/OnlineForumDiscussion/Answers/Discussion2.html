Introduction:

NoSQL is a method of database management that is non-relational or distributed and it is derived from the traditional relational databases. SQL databases are table-based databases where NoSQL can be document stores, key-value stores, graph databases and etc. SQL databases have schemas defined where NoSQL databases use dynamic schemas for unstructured data. SQL databases are vertically scalable in most situations. You’re able to increase the load on a single server by adding more CPU, RAM, or SSD capacity. NoSQL databases are horizontally scalable. You’re able to handle higher traffic by sharding, which adds more servers to your NoSQL database. (Smallcombe, 2021)

Key Value stores is a method used to stored and represent a collection of data. The key is the unique identifier for the respective value which helps keep track of the data. The values can be any type of object -- a number or a string, or even another key-value pair in which case the structure of the database grows more complex (Williams, 2021). They Key acts as the index when searching, updating and deleting the associated value.

Key-value stores are easily scalable, easy to access data, simple to use and easy to move from one system to another without the need for new architecture or changing the code.
