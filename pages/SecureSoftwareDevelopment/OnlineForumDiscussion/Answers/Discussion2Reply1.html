Cryptography case study: TrueCrypt - Reply

Great post Yin, very detailed ontology. The source code for both the bootloader and the Windows kernel driver did not meet expected standards for secure code. This includes issues such as lack of comments, use of insecure or deprecated functions, inconsistent variable types, and so forth. (Junestam -Security and Guigo -Security Engineer, 2014). As the authors of TrueCrypt mention, Using TrueCrypt is not secure as it may contain unfixed security issues (truecrypt.sourceforge.net, n.d.), therefore I would not recommend this software to my family or friends instead I will recommend Bitlocker or Veracrypt (Githinji, n.d.).

References:

Junestam -Security, A. and Guigo -Security Engineer, N. (2014). Open Crypto Audit Project TrueCrypt Security Assessment Prepared for: Prepared by. [online] Available at: https://opencryptoaudit.org/reports/iSec_Final_Open_Crypto_Audit_Project_TrueCrypt_Security_Assessment.pdf.
