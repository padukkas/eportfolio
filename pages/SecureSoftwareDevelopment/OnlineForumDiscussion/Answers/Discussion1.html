Introduction:

An SQL injection attack is when an attacker injects a SQL query via the data input form on an application. If a SQL injection attack was successful, the attacker can read, modify (update, delete, insert and create) data, perform administrative operations in a database and many more malicious activities.

A hacker executes an SQL injection with an SQL statement that is always true. For instance, 1=1; instead of just entering the “wrong” input, the hacker uses a statement that will always be true. Entering “100 OR 1=1” in the query input box will return a response with the details of a table. (SECURITY, n.d.)

Defenses against SQL injection (OWASP, 2019)

  • Use of Prepared Statements (with Parameterized Queries)
  • Use of Stored Procedures
  • Allow-list Input Validation
  • Escaping All User Supplied Input

    • Parameterized queries make sure that the parameters (user inputs) passed into SQL statements are treated in a safe manner.

    Input validation is performed to ensure only properly formed data is entering the workflow in an information system, preventing malformed data from persisting in the database and triggering malfunction of various downstream components.
