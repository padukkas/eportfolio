Faceted Data

Do you think this is a good approach to protect systems from data leakage? What are the pros and cons?.

Yes, I think this approach is more than capable of protecting the system for a data leak because facets, only contains functional units, which are independent of the data. It makes sure traditional information flow security property of termination-insensitive noninterference, independent of any bugs, vulnerabilities, or malicious code in the client application (Schmitz et al., 2016).

When handling user data that needs to manipulated by a developer or by someone else the application must always ensure that data does not get misused on purpose or by accident. Information flow controls require a developer to mark sensitive information, but otherwise automatically prevent any “leaks” of this data (Schmitz et al., 2016).

One of the advantages is that the code is independent of the data making the code more reliable as the code is not written with data inputs or data types in mind. The developer has the system to work with any type of data sets making the system more reliable.

One of the disadvantages I see is the lack of visibility for the developer. When debugging the code direct data injection is going to be difficult near impossible as the facets are segregated from the data.

References:

Schmitz, T., Rhodes, D., Austin, T.H., Knowles, K. and Flanagan, C. (2016). Faceted Dynamic Information Flow via Control and Data Monads. Lecture Notes in Computer Science, 9635, pp.3–23. doi:10.1007/978-3-662-49635-0_1.
