Team Activity Discussion

1. What factors determine whether a programming language is secure or not?

I personally do not believe there is one programming language that is secure if the coding is not done in a secure manner. There are common security vulnerabilities that we can look out for when identifying if a programming language is secure or not.

  • Overflow errors
  • Proper input validations
  • Access control
  • Cryptography handling
  • Encryption techniques
  • Hashing techniques
  • Data manipulation

3. Python would be a better language to create operating systems than C. Discuss.

  • In theory it will be possible, but I think the work to get around obstacles are going to be time consuming. Python does not natively allow control, read and write access to resources like CPU and Memory where C will be able to access those resources.
  • Since OS programming requires low-level system programming python will not be that suitable since python is a high-level programming language. (wiki.osdev.org, 2009)
