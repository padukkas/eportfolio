Great post Ahmed, I agree with your comment regarding automated testing.

Even though automated testing is relatively a new approach to software devlopment it allows companies and development teams to release products, product changes and ehancements with confidence that it will not break the production environment.

Automated tests come in different types, unit testing which helps to ensure the code the developers implement don't break the existing code, then there is UI tests which ensures the interface has not been broken, we also have load testing making sure the application can handle the load. As you mentioned all the above types of automated testing is a key component to continuous integration and continuous deployment.
